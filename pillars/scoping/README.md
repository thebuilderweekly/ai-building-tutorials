# Scoping

Killing bad ideas before building. Tutorials in this pillar teach patterns for using agents to critique, stress-test, and triage product ideas so you don't spend a week shipping something that solves the wrong problem.

## The pattern

Every scoping tutorial follows the same arc: a builder has an idea, the idea hasn't been adversarially critiqued, an agent runs a structured critique battery against the idea, and the builder either sharpens the idea (new scope, new constraints) or kills it entirely before writing any code.

- **Before-state:** "You have a PRD you're excited about. You start building. Four days in, you realize it solves the wrong problem."
- **After-state:** "Before you write any code, an agent runs six adversarial critiques against the PRD and surfaces the three objections that would have killed the idea on day four."

The load-bearing artifact in every scoping tutorial is the prompt set. A tutorial that says "ask Claude to critique your idea" and doesn't give you the six specific prompts to use isn't finished — the value is in the exact battery, not the vague instruction.
